Introduction

This project is representing a list of applications that applicants sent to Czech foreign policy in order to prolong Czech longterm visa. In this sample there are 5 applicants as it was required in the project description. This list could be processed and used by different software, such as an administrating software for the Czech Foreign Ministry. When they want to check some applicants, where they have registered, have they some changes in first name or last name, were some restrictions on them. These things can be showed by processing the data from my XML file. The project is available on https://github.com/vpukha/visaPermissionApp.


Methodology – used technologies

As required in this project, I used:

  • XML
  • XML Schema
  • HTML
  • XSL
  • XSLT

Everything was validated by means of W3C Online Validator


Work content

This project consists of 5 files:

XML file
Comprises the main content of the project
XML Schema file
Describe the structure of the XML document above
XSL Transformation file
Style sheet for XML
HTML file
Contains the transforamtion from XML to an HTML table

Elements description

The root element in XML is <applications>. It contains five children elements called <applicant> with different id's.

Each <applicant> element contains:

Consular office

  • <country>
  • <city>
  • etc.

Basic information

  • <firstname>
  • <lastname>
  • etc.

Additional information

  • <full_placeofbirth>
  • <nationality>
  • etc.

Registration where living

  • <placeofRegistration>
  • <temporaryRegistration>
  • etc.

Purpose and documents

  • <IDforAbroad>
  • <purpuseOfstay>
  • etc.

Additional information

  • <purposeIsProofed>
  • <restrictions>
  • etc.

Transformation description

XSL was used in order to transform XML file data into HTML table is showing a list of applicants sorted by their last names in alphabetic order.

Applicants which have a proofed visa are highlited with green color, others with red.
